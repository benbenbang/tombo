    Real-world Workflows¶

    Learn how to leverage Tombo effectively in common Python development scenarios. These workflows demonstrate practical usage patterns that speed up your daily development tasks.

    New Project Setup¶

    Starting a Web Application¶

    Scenario: Creating a new Django REST API project

    Step 1: Initialize Project Structure 

    mkdir my-api-project
cd my-api-project
python -m venv venv
source venv/bin/activate

    Step 2: Create pyproject.toml with Tombo

    pyproject.toml
    [project]
name = "my-api"
version = "0.1.0"
requires-python = ">=3.11"
dependencies = [
    "django",              # ‚Üê Hover to see latest version
    "djangorestframework", # ‚Üê Use completion for >=
    "psycopg2-binary",     # ‚Üê Add version constraint
]

[project.optional-dependencies]
dev = [
    "pytest",             # ‚Üê Type >= for version completion
    "black",              # ‚Üê Get latest stable version
    "isort",              # ‚Üê Smart version suggestions
]

    Step 3: Tombo-Assisted Completion 1. Type package name ‚Üí Hover shows package info 2. Add constraint operator (>=) ‚Üí Version completion appears 3. Select appropriate version ‚Üí Tombo suggests compatible versions 4. Verify compatibility ‚Üí Hover shows Python version requirements

    Step 1: Audit Current Dependencies

    Step 2: Version Research with Tombo 1. Hover over packages ‚Üí See current vs latest versions 2. Check breaking changes ‚Üí Click documentation links 3. Review compatibility ‚Üí Verify Python version requirements 4. Plan update strategy ‚Üí Prioritize safe vs major updates

    Step 3: Safe Updates First

    Updated pyproject.toml

    Handling Breaking Changes¶

    Scenario: Upgrading Pydantic v1 to v2

    Before Migration:

    pyproject.toml - Before
    dependencies = [
    "pydantic>=1.10.0",    # ‚Üê Hover shows v2.4.0 available
    "fastapi>=0.104.0",    # ‚Üê Check Pydantic v2 compatibility
]

    Research Process: 1. Hover on pydantic ‚Üí See v2 is major update 2. Click documentation link ‚Üí Read migration guide 3. Hover on fastapi ‚Üí Verify v2 compatibility 4. Check dependencies ‚Üí Review impact on other packages

    After Migration:

    pyproject.toml - After
    dependencies = [
    "pydantic>=2.4.0",     # ‚Üê Updated with breaking changes
    "fastapi>=0.104.0",    # ‚Üê Confirmed compatible
]

    6. Click PyPI links ‚Üí Check package maintenance status
    7. Review release dates ‚Üí Avoid abandoned packages

    8. Check for yanked versions ‚Üí Ensure stable version selection

    9. Impact Assessment

    11. Check documentation links ‚Üí Verify package quality
    12. Review constraint ranges ‚Üí Avoid future conflicts

    Docker Integration¶

    Scenario: Containerized application with optimized builds

    Multi-stage Dockerfile:

    Dockerfile
    # Build stage - includes development tools
FROM python:3.11-slim as builder
COPY pyproject.toml .
# Tombo helps ensure these versions work together
RUN pip install build setuptools wheel

# Production stage - only runtime dependencies
FROM python:3.11-slim as production
COPY --from=builder /app/dist/*.whl .
RUN pip install *.whl

# Development stage - includes dev tools
FROM production as development
RUN pip install -e .[development]

    Benefits of Tombo Integration: - Version research ‚Üí Hover to find stable versions for containers - Compatibility verification ‚Üí Ensure Python base image compatibility - Documentation access ‚Üí Quick links to installation guides - Update planning ‚Üí Easy to assess update impact before rebuild

    Django==3.2.19
psycopg2-binary==2.8.6
requests==2.25.1
pytest==6.2.4

    Migration Process with Tombo:

    1. Create pyproject.toml structure
    2. Research current versions - Hover over each package
    3. Check breaking changes - Click documentation links
    4. Plan update strategy - Group by risk level

name = "legacy-app"

    "psycopg2-binary>=2.9.0", # ‚Üê Security updates

[project.optional-dependencies]
dev = [
    "pytest>=7.4.0",          # ‚Üê Major version upgrade
    "black>=23.0.0",          # ‚Üê Latest stable formatter
]

    Migration Benefits: - Informed updates ‚Üí Tombo hover shows what's changed - Gradual migration ‚Üí Update packages incrementally - Compatibility checking ‚Üí Verify Python version requirements - Documentation access ‚Üí Read migration guides directly

    CI/CD Pipeline¶

    Scenario: Automated dependency management in GitHub Actions

    .github/workflows/dependencies.yml
    name: Dependency Management
on:
  schedule:
    - cron: '0 9 * * 1'  # Weekly Monday morning

jobs:
  update-deps:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: pip install -e .[dev]

      - name: Run tests
        run: pytest

      # Manual dependency research with Tombo after automated checks
      - name: Create dependency report
        run: |
          echo "Review dependencies with Tombo in VS Code"
          echo "- Hover over each package for update information"
          echo "- Check documentation links for breaking changes"
          echo "- Use version completion for upgrade planning"

    Pre-commit Integration¶

    Scenario: Ensuring dependency consistency in team commits

    .pre-commit-config.yaml
    repos:
  - repo: local
    hooks:
      - id: check-dependencies
        name: Check dependency versions
        entry: python scripts/check_deps.py
        language: system
        files: pyproject.toml
    scripts/check_deps.py
    #!/usr/bin/env python3
"""
Pre-commit hook to validate dependencies.
Use Tombo in VS Code for manual verification of flagged packages.
"""
import tomllib
import sys

def check_dependencies():
    with open("pyproject.toml", "rb") as f:
        data = tomllib.load(f)

    deps = data.get("project", {}).get("dependencies", [])

    # Flag packages that might need attention
    attention_needed = []
    for dep in deps:
        if ">=" not in dep and "~=" not in dep and "==" not in dep:
            attention_needed.append(dep)

    if attention_needed:
        print("‚ö†Ô∏è  Dependencies without version constraints:")
        for dep in attention_needed:
            print(f"   - {dep}")
        print("\nüí° Use Tombo in VS Code to add appropriate version constraints")
        return 1

    return 0

if __name__ == "__main__":
    sys.exit(check_dependencies())

    Best Practices Summary¶

    Daily Development¶

    1. Start with hover ‚Üí Understand packages before using
    2. Use completion actively ‚Üí Let Tombo suggest appropriate versions
    3. Check compatibility ‚Üí Verify Python version requirements
    4. Read documentation ‚Üí Click links for detailed package info
    5. Update gradually ‚Üí Use hover to assess update safety

    Team Workflows¶

    1. Standardize constraints ‚Üí Agree on constraint types (~= vs >=)
    2. Document decisions ‚Üí Use comments to explain version choices
    3. Review dependencies ‚Üí Use Tombo during code review process
    4. Share knowledge ‚Üí Discuss package alternatives found via research
    5. Monitor security ‚Üí Check for yanked versions and security updates

    Production Deployments¶

    1. Pin critical dependencies ‚Üí Use == for stability where needed
    2. Test thoroughly ‚Üí Verify updates in staging environments
    3. Plan rollback strategy ‚Üí Keep working versions documented
    4. Monitor performance ‚Üí Check for regressions after updates
    5. Automate safely ‚Üí Combine automated tools with manual verification
